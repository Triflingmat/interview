## 一、CSS 核心高频考点

### 1. 水平 / 垂直 / 水平垂直居中的 N 种方法

#### 水平居中

1. 行内元素：`text-align: center`
2. 定宽块级元素：`margin: 0 auto`
3. 弹性布局：父元素 `display: flex; justify-content: center`

#### 垂直居中

1. 单行行内元素：`line-height = 元素高度`
2. 弹性布局：父元素 `display: flex; align-items: center`
3. 绝对定位：`position: absolute; top: 50%; transform: translateY(-50%)`

#### 水平垂直居中（最常用）

1. **Flex 方案**（推荐）：父元素 `display: flex; justify-content: center; align-items: center`
2. **绝对定位 + Transform**：`position: absolute; top:50%; left:50%; transform: translate(-50%,-50%)`
3. **绝对定位 + margin**：定宽高，`position: absolute; top/left/right/bottom:0; margin:auto`
4. Grid 方案：父元素 `display: grid; place-items: center`

### 2. CSS 选择器及优先级

1. **优先级公式**：`!important` > 行内样式 > ID 选择器 > 类选择器 / 属性选择器 / 伪类 > 元素选择器 / 伪元素 > 通配符`*`
2. **权重计算**：行内 (1000) > ID (100) > 类 / 伪类 (10) > 元素 (1)
3. **核心规则**：权重相同取**就近原则**，继承样式优先级最低。

### 3. Flex 弹性布局核心属性

#### 容器属性（父元素）

- `display: flex`：开启弹性布局
- `flex-direction`：主轴方向（row/column/row-reverse）
- `justify-content`：主轴对齐方式
- `align-items`：交叉轴对齐方式
- `flex-wrap`：是否换行（nowrap/wrap）
- `align-content`：多行交叉轴对齐

#### 项目属性（子元素）

- `flex-grow`：放大比例
- `flex-shrink`：缩小比例
- `flex-basis`：初始宽度
- `align-self`：单独设置交叉轴对齐

### 4. 盒子模型 + BFC

#### 盒子模型

1. **标准盒模型**（默认）：`box-sizing: content-box`，`width = 内容宽度`
2. **IE 盒模型**：`box-sizing: border-box`，`width = 内容+内边距+边框`（开发首选）

#### BFC（块级格式化上下文）

1. 定义：独立的渲染区域，内部元素不影响外部
2. **触发条件**：`overflow: hidden`、`float` 非 none、`position: absolute/fixed`、`display: flex/grid`
3. **应用场景**：清除浮动、解决 margin 塌陷、防止父元素高度塌陷

### 5. position 定位的区别

1. `static`：默认值，无定位
2. `relative`：相对自身定位，**不脱离文档流**，保留原位置
3. `absolute`：相对最近非 static 父级定位，**脱离文档流**
4. `fixed`：相对视口定位，固定不随滚动，脱标
5. `sticky`：粘性定位，滚动到阈值后固定，结合 relative+fixed

### 6. 响应式布局实现方式

- 媒体查询：`@media (max-width:768px){}` 适配不同屏幕
- rem：相对根节点`html`的`font-size`
- vw/vh：视口单位，1vw=1% 视口宽度
- 弹性布局 Flex、网格布局 Grid、百分比布局

### 7. em 和 rem 的区别

1. `em`：相对**父元素**的字体大小，嵌套层级多易混乱
2. `rem`：相对**根节点 html**的字体大小，统一规范，是响应式布局首选

## 二、JavaScript (含 TypeScript) 核心考点

### 1. 事件循环 (Event Loop)

1. JS 是**单线程**语言，为解决异步阻塞设计事件循环
2. **宏任务**：script 整体、setTimeout、setInterval、IO、UI 渲染
3. **微任务**：Promise.then/catch、async/await、MutationObserver
4. **执行顺序**：同步代码 → 清空当前微任务队列 → 执行下一个宏任务

### 2. 异步方案：Promise 和 async/await

1. **Promise**：解决回调地狱，三种状态（pending/fulfilled/rejected），支持链式调用
2. **async/await**：Promise 的**语法糖**，用同步写法写异步代码
3. **区别**：Promise 链式调用可读性一般；async/await 更简洁，错误捕获用`try/catch`

### 3. 原型与继承

#### 原型链

- 每个对象有`__proto__`，指向构造函数的`prototype`
- 原型链顶端：`Object.prototype.__proto__ = null`
- 查找规则：先找自身属性，找不到沿原型链向上查找

#### new 一个对象的过程

1. 创建空对象
2. 绑定原型：`obj.__proto__ = 构造函数.prototype`
3. 执行构造函数，`this`指向新对象
4. 返回新对象

### 4. 作用域与闭包

#### this 绑定规则

1. 默认绑定：全局环境指向 window（严格模式 undefined）
2. 隐式绑定：对象调用指向该对象
3. 显式绑定：`call/apply/bind` 强制指定 this
4. new 绑定：指向 new 创建的实例
5. 箭头函数：**无自己的 this**，继承外层作用域的 this

#### var/let/const 区别

|   特性   |  var   |     let/const      |
| :------: | :----: | :----------------: |
|  作用域  | 函数级 |        块级        |
| 变量提升 |   有   |  有（暂时性死区）  |
| 重复声明 |  允许  |       不允许       |
|   赋值   | 可修改 | const 不可修改引用 |

#### call/apply/bind

- `call/apply`：立即执行函数，改变 this 指向；call 参数列表，apply 数组传参
- `bind`：返回一个新函数，**永久绑定 this**，不立即执行

### 5. ES6+ 新特性

1. Set/Map vs Object
   - Set：无序不重复的值集合
   - Map：键值对，键可以是任意类型，有序，无原型链污染
   - Object：键只能是字符串 / Symbol，无序，有默认原型属性
2. 其他核心：箭头函数、解构赋值、扩展运算符、Promise、class、模块化

### 6. TypeScript 核心

1. **TS 的好处**：静态类型校验，提前发现错误；提升代码可维护性；IDE 智能提示；适配大型项目
2. interface vs type
   - interface：定义对象 / 类，支持继承、合并
   - type：定义任意类型（基本 / 联合 / 元组），不支持合并
3. **泛型**：不预先指定类型，使用时传入，实现代码复用 + 类型安全

### 7. 深浅拷贝

1. 浅拷贝

   ：只复制第一层，引用类型共享内存地址

   - 实现：`Object.assign`、扩展运算符`...`、数组`slice`

2. 深拷贝

   ：递归复制所有层级，引用类型独立

   - 实现：`JSON.parse(JSON.stringify)`、递归函数、`lodash.cloneDeep`

3. **区别**：浅拷贝修改原对象会影响拷贝对象；深拷贝完全独立

### 8. 数组常用方法

- `map`：遍历返回**新数组**
- `filter`：过滤符合条件的元素，返回新数组
- `reduce`：累加 / 计算，返回**单个值**
- `forEach`：纯遍历，无返回值
- `find`：返回第一个匹配的元素

## 三、浏览器和网络安全 综合考点

### 1. 从输入 URL 到页面展示全流程

1. DNS 解析：域名 → IP 地址
2. TCP 三次握手：建立连接
3. 发送 HTTP/HTTPS 请求
4. 服务器响应，返回资源
5. TCP 四次挥手：断开连接
6. 浏览器解析：构建 DOM 树 + CSSOM 树 → 合成渲染树
7. 布局（回流）→ 绘制（重绘）→ 图层合成 → 页面展示

### 2. 跨域

1. **同源策略**：协议 + 域名 + 端口完全相同，否则跨域
2. 解决方案
   - CORS（首选）：后端配置响应头
   - 代理服务器：webpack devServer、nginx 反向代理
   - JSONP：仅支持 GET 请求
   - postMessage、iframe

### 3. 浏览器缓存

1. 强缓存

   ：不发送请求，直接读取缓存

   - 状态码：200（from cache）
   - 字段：`Cache-Control`、`Expires`

2. 协商缓存

   ：发送请求，服务器验证资源未修改返回 304

   - 字段：`Last-Modified/If-Modified-Since`、`ETag/If-None-Match`

3. 优先级：强缓存 > 协商缓存

### 4. 本地存储对比

|      类型      | 大小 |        生命周期        | 随请求发送 |
| :------------: | :--: | :--------------------: | :--------: |
|     cookie     | 4KB  |     可设置过期时间     |     是     |
|  localStorage  | 5MB  |        永久存储        |     否     |
| sessionStorage | 5MB  | 会话级（关闭标签失效） |     否     |

### 5. HTTP 核心

1. 常用状态码
   - 200：成功
   - 301/302：重定向；304：缓存命中
   - 400：参数错误；401：未授权；403：禁止访问；404：资源不存在
   - 500：服务器错误
2. GET vs POST
   - GET：查询数据，参数在 URL，长度限制，幂等
   - POST：提交数据，参数在 body，无限制，非幂等

### 6. HTTPS 加密原理

1. HTTPS = HTTP + SSL/TLS 加密
2. **对称加密**：速度快，密钥传输不安全
3. **非对称加密**：公钥加密、私钥解密，安全但速度慢
4. **流程**：服务器下发数字证书（含公钥）→ 客户端验证证书 → 生成随机密钥用公钥加密 → 服务器解密 → 对称加密传输数据

### 7. 安全攻击：XSS 和 CSRF

1. XSS（跨站脚本）
   - 原理：注入恶意脚本，窃取用户数据
   - 防御：输入转义、CSP、HttpOnly Cookie
2. CSRF（跨站请求伪造）
   - 原理：利用用户 Cookie，伪造请求
   - 防御：Token 验证、SameSite Cookie、校验 Referer

### 8. HTTP 版本区别

- HTTP1.0：短连接，无多路复用
- HTTP1.1：长连接，队头阻塞
- HTTP2.0：多路复用、二进制分帧、头部压缩、服务器推送

### 9. Token 登录鉴权流程

1. 用户输入账号密码 → 服务器验证通过
2. 生成 JWT Token 返回给客户端
3. 客户端存储 Token（localStorage）
4. 每次请求在请求头携带`Authorization: Bearer Token`
5. 服务器验证 Token，返回数据

### 10. CDN 加速原理

内容分发网络，将静态资源缓存到全球节点，用户就近访问，降低延迟，减轻源服务器压力。

### 11. SSR vs CSR

1. **CSR（客户端渲染）**：首屏慢、白屏、SEO 差，服务器压力小
2. **SSR（服务端渲染）**：首屏快、SEO 友好，服务器压力大

## 四、Vue 框架核心考点

### 1. 核心对比类

1. Vue vs React
   - Vue：双向绑定、模板语法、上手简单
   - React：单向数据流、JSX、生态更丰富
2. Vue2 vs Vue3
   - Vue3：Proxy 响应式、组合式 API、性能提升、TS 原生支持
3. Pinia vs Vuex
   - Pinia：无 mutation、TS 友好、体积小、语法简洁（Vue3 首选）
4. Webpack vs Vite
   - Webpack：兼容性强，打包慢
   - Vite：开发热更新极快，基于 ESBuild，生产用 Rollup

### 2. 响应式原理

1. Vue2：

   ```
   Object.defineProperty
   ```

    劫持对象属性

   - 缺陷：无法监听新增 / 删除属性、数组下标 / 长度修改

2. Vue3：

   ```
   Proxy
   ```

    代理整个对象

   - 优势：监听所有操作、支持数组、性能更好

### 3. 组件通信方式

- 父子组件：`props/$emit`、`v-model`
- 跨级组件：`provide/inject`
- 兄弟 / 全局：`EventBus`、`Pinia/Vuex`

### 4. 虚拟 DOM 与 Diff 算法

1. **虚拟 DOM**：用 JS 对象描述真实 DOM，减少频繁 DOM 操作，提升性能
2. **key 的作用**：唯一标识节点，Diff 算法复用节点，避免错误渲染
3. **Diff 核心**：同层比较、深度优先、key 复用，最小化 DOM 操作

### 5. 其他核心

1. **Tree Shaking**：基于 ES 模块，移除未使用代码，减小打包体积
2. **生命周期**：Vue3 用`setup`替代 beforeCreate/created，组合式 API（onMounted 等）
3. **nextTick**：DOM 更新完成后执行回调，获取最新 DOM
4. **keep-alive**：缓存组件，避免重复渲染，触发`activated/deactivated`
5. **MVVM**：Model（数据）-View（视图）-ViewModel（双向绑定，同步数据和视图）

## 五、前端工程化

### 1. Webpack 核心概念

- Entry：打包入口
- Output：打包输出目录
- Loader：编译非 JS 文件（css-loader、babel-loader）
- Plugin：扩展功能（HtmlWebpackPlugin、TerserPlugin）
- 流程：解析入口 → 依赖分析 → Loader 编译 → Plugin 处理 → 输出产物

### 2. Webpack 优化方案

- **打包速度**：多线程、开启缓存、缩小文件搜索范围、ESBuild 替换 Babel
- **产物体积**：Tree Shaking、代码分割、资源压缩、图片优化、CDN

### 3. 模块化

- **CommonJS**：Node 专用，`require`同步加载，运行时解析
- **ESM**：浏览器 / Node 通用，`import/export`，静态编译，编译时解析

## 六、性能优化

### 1. 性能优化方法论

1. **网络层面**：HTTP2、CDN、浏览器缓存、资源压缩、懒加载
2. **构建层面**：Webpack 优化、Tree Shaking、代码分割
3. **代码层面**：减少 DOM 操作、防抖节流、虚拟列表、图片优化

### 2. 白屏问题排查

- 网络：资源加载缓慢、请求失败
- 渲染：JS 阻塞渲染、CSS 阻塞、首屏资源过大
- 框架：CSR 无 SSR、组件渲染异常

### 3. 内存泄漏排查

- 控制台 Memory 快照分析
- 常见原因：未清除定时器、闭包滥用、DOM 引用未释放、事件监听未移除

### 4. 性能指标与埋点

- 核心指标：FP（首次绘制）、FCP（首次内容绘制）、LCP（最大内容绘制）、TTI（可交互时间）
- 埋点方案：手动埋点、可视化埋点、全埋点，上报数据做监控
